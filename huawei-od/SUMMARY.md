# 华为 OD 机考题型归纳与解法沉淀

这个模块收录了华为 OD（Outsourcing Development）机考高频真题，按算法模型分类归纳。目标是把机考中反复出现的题型模板沉淀下来，遇到相似题目能快速套用。

## 目录结构

`com.xujn.od` 包下按题目名称扁平组织，75 道题覆盖 DP、BFS/DFS、贪心、模拟、字符串、背包、并查集、图论等核心考点。

## 一、题型分类与共性解法

### 1. 动态规划（DP）—— 出现频率最高的题型

识别信号：求"最大/最小/方案数"，且问题可以拆成重叠子问题。

| 子类型 | 代表题目 | 状态定义 | 核心转移 |
|--------|----------|----------|----------|
| 路径 DP | 流浪地球、亲子游戏 | dp[i][j] = 到 (i,j) 的最大和 | dp[i][j] = max(上, 左) + grid[i][j] |
| 线性 DP | 跳房子 | dp[i] = 到第 i 格最大收益 | dp[i] = max(dp[j]) + coins[i], j∈[i-k,i) |
| 打家劫舍 | 编码能力提升 | dp[i][0/1] = 第 i 天做/不做 | dp[i][1] = dp[i-1][0] + v[i] |
| 背包变形 | 游戏分组、贪心歌手、购物 | dp[j] = 容量 j 时最优值 | dp[j] = max(dp[j], dp[j-w]+v) |
| 子集和 | 网上商城优惠、任务最优调度 | dp[j] = true/false 能否凑出 | dp[j] |= dp[j-items[i]] |
| 票价 DP | Wonderland | dp[i] = 到第 i 天最小花费 | dp[i] = min(dp[i-1]+c1, dp[i-7]+c7, ...) |

共性模板（0-1 背包）：
```
for 每个物品 i:
    for j 从容量 倒序到 weight[i]:
        dp[j] = max(dp[j], dp[j - weight[i]] + value[i])
```

面试关注点：
- 是否分清"0-1 背包"（每个物品只能选一次，j 倒序）和"完全背包"（j 正序）
- 最小差分割 → 目标是让一组之和尽量接近 totalSum/2
- 子集和 → dp 存 boolean，不是 max

### 2. BFS / DFS / 图搜索 —— 第二高频

识别信号：网格地图、连通区域、最短步数、层级关系。

| 子类型 | 代表题目 | 核心模型 |
|--------|----------|----------|
| 网格 BFS | 智能驾驶、矩阵扩散 | 四方向扩展 + visited 数组 |
| 多源 BFS | 矩阵扩散 | 初始把所有源点加入队列 |
| 连通分量 | 寻找最大价值矿堆 | BFS/DFS 标记连通区域求和 |
| 带边界条件 | 查找单入口空闲区域 | BFS + 额外统计边界接触数 |
| 最短路 BFS | 乘坐保密电梯 | 状态空间 BFS（楼层=节点） |
| Dijkstra | 路口最短时间 | 优先队列 + 松弛操作 |
| 拓扑排序 | 微服务集成测试、模块执行顺序 | Kahn BFS 层序 + 环检测 |
| 二叉树 DFS | 传递悄悄话 | 递归到叶子求最大路径和 |
| 矩阵搜索 | 找到它 | DFS + 回溯搜索单词 |

共性模板（网格 BFS）：
```
queue.offer(起点); visited[起点] = true;
while queue 非空:
    pos = queue.poll()
    for 四方向:
        if 合法 && 未访问 && 满足条件:
            visited = true; queue.offer(新位置)
```

面试关注点：
- 多源 BFS 和单源 BFS 的区别仅在于初始化
- 拓扑排序的"环检测" → 最终入度为 0 的节点数 < 总节点数
- Dijkstra 只适用于非负权边

### 3. 贪心算法

识别信号：局部最优选择可以推出全局最优，通常涉及排序。

| 子类型 | 代表题目 | 贪心策略 |
|--------|----------|----------|
| 任务调度 | 高效的任务规划 | 按截止时间升序排列 |
| 反悔贪心 | 执行任务赚积分 | 小顶堆维护已选项，替换最差的 |
| 区间覆盖 | 路灯照明、敌情监控 | 排序后按右端点贪心选取 |
| 区间合并 | 路灯照明 | 排序后合并重叠区间 |
| 哈夫曼 | 生成哈夫曼树 | 每次合并最小的两个 |
| 最优分配 | 采购订单 | 每类选最便宜的 |

面试关注点：
- 贪心题最难的是**证明贪心策略正确**（交换论证法）
- "反悔贪心" = 先贪心选，发现更优的再替换（用堆维护）
- 区间问题几乎都需要先排序

### 4. 字符串处理

识别信号：解析、匹配、替换、滑动窗口。

| 子类型 | 代表题目 | 核心技巧 |
|--------|----------|----------|
| 表达式计算 | 最长数学表达式 | 栈处理优先级（* 先算，±入栈最后求和） |
| 回文 | 对称字符串 | 中心扩展法 O(n²) |
| 滑动窗口 | 获得完美走位 | 维护窗口外各方向计数 ≤ target |
| 递归解析 | 仿 LISP 运算 | 递归下降：遇 `(` 递归，遇 `)` 收拢 |
| 正则/替换 | 日志脱敏 | `replaceAll("\\d+", "***")` |
| 前缀匹配 | IMSI 匹配 | 按前缀长度降序，首个匹配即最长 |

### 5. 模拟与数据结构设计

识别信号：题目直接描述了一套操作规则，按规则逐步执行。

| 子类型 | 代表题目 | 核心数据结构 |
|--------|----------|-------------|
| 队列模拟 | 模拟消息队列 | LinkedList 做 Queue |
| LRU 缓存 | 文件缓存系统 | LinkedHashMap(accessOrder=true) |
| 内存管理 | 简易内存池 | TreeMap (First Fit 分配) |
| 任务调度 | 模拟工作队列 | 优先队列（完成时间+机器编号） |
| 扫描线 | 池化资源共享 | 事件排序 +1/-1 求最大并发 |
| 并查集 | 关联端口组合并 | Union-Find 合并有交集的集合 |

### 6. 数学与位运算

| 子类型 | 代表题目 | 核心公式/技巧 |
|--------|----------|--------------|
| 组合数 | 分苹果 | C(m-1, n-1) 隔板法 |
| 康托展开 | 第 K 个排列 | k / (n-1)! 确定每位数字 |
| Varint 编码 | 整数编码 | 每 7 位分组 + 最高位标记 |
| 不等式求解 | 不等式约束 | 收集所有下界和上界取交集 |

## 三、算法模型速查表

遇到题目不知道用什么算法？按以下线索快速定位：

| 题目特征 | 优先尝试 |
|----------|----------|
| "最少/最多/最大/最小" + 选择方案 | 动态规划 或 贪心 |
| 网格地图 + 最短步数 | BFS |
| 网格地图 + 连通区域 | BFS/DFS 连通分量 |
| "所有方案/排列/组合" | 回溯（DFS + 剪枝） |
| 有向图 + 执行顺序 | 拓扑排序 |
| 加权最短路 | Dijkstra |
| "分成两组差最小" | 0-1 背包（target = sum/2） |
| "容量限制下最大价值" | 0-1 背包 |
| "合并有交集的集合" | 并查集 |
| "按规则操作" + 明确步骤 | 模拟 |
| "同一时刻最大并发" | 扫描线 / 差分数组 |
| "前缀/子串匹配" | Trie / 哈希 |
| "括号/嵌套结构" | 栈 |
| "区间覆盖/选取" | 排序 + 贪心 |
