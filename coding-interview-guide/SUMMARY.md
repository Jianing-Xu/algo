# 程序员代码面试指南题型归纳

这个模块按《程序员代码面试指南》的高频题型组织代码，当前共有 65 个源码文件。目标不是机械抄书，而是把面试里最常被追问的模型先落成可运行代码。

## 目录结构

- `stackqueue`：栈、队列、单调队列、最大矩形、可见山峰、猫狗队列、MaxTree、窗口统计、中位数、最近更小
- `linkedlist`：链表反转、回文判断、环检测、链表分区、随机指针复制
- `binarytree`：非递归遍历、序列化与反序列化、LCA、最大 BST 子树、zigzag 层序、重建、后继节点、完全树计数、修复 BST、边界打印
- `recursiondp`：递推优化、动态规划、换钱、LIS、编辑距离、打气球、N 皇后、汉诺塔、博弈型 DP、LCS、公共子串、布尔表达式求值、字符串交错
- `string`：旋转词、滑动窗口
- `arraymatrix`：转圈打印、最大子数组和、子矩阵最大和、最长累加和、TopK、BFPRT、自然数排序、奇偶调整、LIS 恢复、旋转数组、局部最小、边界二分
- `verifier`：对数器、随机数据生成、暴力解校验

## 1. 栈和队列

核心特征：题目表面在考数据结构设计，实际上常考“辅助结构是否能同步维护信息”。

- `GetMinStack`：额外维护一个同步最小栈，把 `getMin()` 降到 `O(1)`。
- `QueueByStacks`：一个栈负责入队，一个栈负责出队；只有出队栈为空时才搬运。
- `SlidingWindowMaximum`：单调双端队列维护候选最大值，典型窗口模型。
- `MaximalRectangle`：将二维矩阵逐行压成柱状图，再套单调栈。
- `VisibleMountainPairs`：环形结构 + 单调栈，处理重复高度时要合并计数。
- `CatDogQueue`：两个队列加时间戳，分别维护局部顺序和全局顺序。
- `MaxTree`：每个节点挂到左右第一个更大值中较小的那个下面，本质是笛卡尔树。
- `SubarrayCountWithBoundedDiff`：双端队列同时维护窗口最大值和最小值。
- `NearLessElements`：单调栈一次遍历求左右最近更小元素，重复值用链表分组。
- `SlidingWindowMedian`：两个有序集合维护窗口左半区和右半区。

面试关注点：

- 是否能说清楚为什么每个元素最多进队和出队一次。
- 是否能处理空结构、重复值、窗口边界。
- 能否把二维问题压缩成一维单调栈模型。
- 面向对象题里是否能把“业务规则”和“出队顺序”拆清楚。
- 遇到“区间最大值/最小值 + 统计”时能否想到双端队列维护窗口。
- 单调栈除了求面积，还常考“最近更大/更小”和窗口统计。

## 2. 链表

核心特征：链表题本质是指针重连，重点是“断点保护”和“结构恢复”。

- `ReverseLinkedList`：三指针完成原地反转。
- `PalindromeLinkedList`：快慢指针找中点，反转右半区，比对后恢复原链表。
- `LinkedListCycleDetector`：Floyd 快慢指针同时解决判环和找入环点。
- `LinkedListPartition`：三段链表拼接，稳定保留原相对顺序。
- `CopyRandomList`：节点穿插复制，避免额外哈希表空间。

面试关注点：

- 反转时是否先保存 `next`。
- 回文判断是否只用 `O(1)` 额外空间。
- 检查结束后能否把链表恢复，避免副作用。

## 3. 二叉树

核心特征：递归版通常不难，真正面试高频是非递归写法和“树 <-> 字符串”的转换。

- `BinaryTreeTraversal`：统一覆盖先序、中序、后序的栈解法。
- `BinaryTreeCodec`：先序序列化，用 `#` 表示空节点，可无损重建。
- `LowestCommonAncestor`：后序信息汇总模型。
- `BiggestBinarySearchSubtree`：树形 DP，返回“最大 BST 头节点 + 子树范围 + 是否 BST”。
- `ZigzagLevelOrderTraversal`：双端队列实现层序方向切换。
- `RebuildBinaryTree`：利用先序定位根节点，用中序切分左右子树。
- `InOrderSuccessor`：分“有右子树”和“向上找第一个把自己当左孩子的祖先”两种情况。
- `CompleteBinaryTreeNodeCounter`：比较左右子树高度，跳过满二叉树部分。
- `LongestPathSum`：前缀和 + 层号哈希表，适合“从上往下”的路径统计。
- `MaxDistanceInTree`：标准树形 DP，同时返回高度和最大距离。
- `TopologyContainment`：判断是否包含另一棵树的全部拓扑结构，不要求是完整子树。
- `LevelOrderCodec`：层序序列化更接近工程里的广度优先存储。
- `RecoverBinarySearchTree`：中序遍历定位逆序对，恢复被交换的两个节点。
- `RebuildBinaryTreeByInPost`：和前序+中序重建互补。
- `BoundaryNodePrinter`：把左边界、叶子、右边界拆开处理。

面试关注点：

- 非递归中序是否能熟练写出“先一路压左，再弹栈访问”的骨架。
- 序列化时是否保留空节点，否则无法唯一反序列化。
- 树形 DP 题里返回信息是否足够且不冗余。
- 完全二叉树计数能否从 `O(n)` 优化到 `O(log^2 n)`。

## 4. 递归与动态规划

核心特征：先写暴力递归，再提炼出状态定义、转移方程和空间压缩。

- `FibonacciSeries`：展示从递归模型到迭代递推的优化。
- `MinPathSum`：二维 DP 压缩成一维数组，保留 `O(cols)` 空间复杂度。
- `MinCoins`：完全背包模型，求最少张数。
- `LongestIncreasingSubsequence`：`ends[] + 二分` 将复杂度降到 `O(n log n)`。
- `EditDistance`：经典二维 DP，统一插入、删除、替换三种操作。
- `BurstBalloons`：区间 DP，枚举“最后打爆哪一个气球”。
- `NQueens`：既有普通回溯版，也有位运算优化版。
- `HanoiStateStep`：根据当前状态反推它是否在最优路径上，以及是第几步。
- `CardGameWinner`：先手/后手双表 DP，典型博弈模型。
- `CoinWays`：完全背包计数模型。
- `LongestCommonSubsequence`：二维 DP 基础母题。
- `BooleanExpressionWays`：区间 DP，左右真假组合按运算符合并。
- `StringInterleaving`：二维可达性 DP。
- `LongestCommonSubstring`：与 LCS 不同，必须连续。
- `EditCost`：编辑距离的代价扩展版。

面试关注点：

- 状态定义是否准确。
- 初始化是否完整。
- 能否继续优化空间复杂度。

## 5. 字符串

核心特征：字符串题大多可以转成数组扫描，常见模型是匹配、窗口和下标映射。

- `IsRotation`：旋转词判断转成 `s1+s1` 是否包含 `s2`。
- `LongestUniqueSubstring`：记录字符上次出现位置，滑动窗口维护无重复区间。

面试关注点：

- 遇到重复字符时，左边界不能回退。
- 需要明确是返回长度、下标还是具体子串。

## 6. 数组与矩阵

核心特征：要先识别遍历顺序或局部最优能否累积成全局最优。

- `SpiralOrderPrinter`：按四条边分层收缩。
- `SubarrayMaxSum`：Kadane 算法，决定“以当前位置结尾的最好答案”。
- `MaxSubmatrixSum`：枚举行边界后压缩列，再做一维 Kadane。
- `LongestSubarraySumEqualsK`：前缀和第一次出现的位置决定最长长度。
- `TopKElements`：小根堆维护最大的 `k` 个数。
- `BfprtSelector`：中位数的中位数保证线性时间选择第 `k` 小。
- `NaturalNumberSorter`：利用值域特征原地归位。
- `OddEvenAdjuster`：双指针把元素放到目标奇偶下标。
- `LisSequenceRecovery`：在 LIS 长度 DP 基础上回溯出一条实际序列。
- `RotatedArrayMin`：旋转数组上的二分缩区间。
- `LocalMinimum`：利用两侧趋势不同定位局部最小。
- `BinarySearchVariants`：第一等于、最后等于、第一个大于等于、最后一个小于等于。

面试关注点：

- 单行、单列、空矩阵边界是否正确。
- 最大子数组和全负数场景是否成立。
- 前缀和类题目是否意识到“最早出现的位置”才最有价值。
- TopK 和第 k 小问题是否能区分堆解法、快排选择、BFPRT 的适用场景。
- 原地数组调整类题，关键通常不是排序，而是“当前位置应该放谁”。

## 7. 对数器

左程云体系里对数器非常关键。核心思路不是“写完样例就算过”，而是：

1. 写一个你确信正确但可能较慢的暴力解。
2. 写一个正式面试解法。
3. 随机生成大量样本，持续比较两个答案。
4. 一旦出现不一致，立即打印样本定位 bug。

这个模块里的 `verifier` 包已经把基础骨架抽出来：

- `RandomCaseGenerator`：负责随机数组、随机字符串、随机参数。
- `AlgorithmComparator`：统一执行“随机样本 -> 暴力解 -> 优化解 -> 比对”流程。
- `AlgorithmComparatorTest`：示范了三个典型对数器场景：
- `AlgorithmComparatorTest`：示范了三个典型对数器场景：
  - 滑动窗口最大值：单调队列 vs 暴力枚举窗口。
  - 最长无重复子串：滑动窗口 vs 双重循环枚举。
  - 最大子数组和：Kadane vs 所有区间枚举。
- `MoreComparatorTest`：再补两个更偏技巧型的场景：
  - 可见山峰：单调栈解法 vs 两点可见性暴力枚举。
  - 最大矩形：柱状图压缩 + 单调栈 vs 枚举所有子矩形。
- `BookComparatorTest`：新增四类更贴近书中题型的对数器：
  - `max-min<=num` 子数组数量：窗口优化 vs 暴力枚举。
  - 子矩阵最大和：二维压缩 vs 枚举所有子矩阵。
  - 累加和等于 `k` 的最长子数组：前缀和哈希 vs 双循环。
  - BFPRT：线性选择 vs 排序取第 `k` 小。

面试和刷题里的实际价值：

- 对边界条件特别有效，尤其是空数组、重复值、负数、极小窗口。
- 适合验证贪心、单调栈、动态规划、树形递归这类容易“看着对、实际上错”的题。
- 可以显著降低重构或优化后的回归风险。

4. 递归与 DP：已覆盖换钱、LIS、LCS、最长公共子串、布尔表达式求值、打气球、N 皇后、汉诺塔状态、博弈型 DP、字符串交错、编辑距离及其代价变体。
5. 数组与矩阵：已覆盖最大子数组、子矩阵最大和、最长累加和、TopK、BFPRT、自然数排序、奇偶调整、LIS 恢复、旋转数组、局部最小、二分边界问题。

## 后续扩展建议

如果继续向更完整的书中题库推进，优先补下面几类：

1. 字符串专题：KMP/BM 实战题、回文串分割、最小包含子串。
2. 链表专题：按值删除、两两交换、k 组翻转等工程常见变体。
3. 树专题：Morris 遍历、按距离统计节点、树上路径问题变体。
4. 数组专题：循环数组问题、单调队列更多计数模型、二维前缀和变体。
